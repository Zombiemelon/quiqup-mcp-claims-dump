<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → Next.js MCP route | Untrusted inbound. Clerk OAuth at+jwt is the only credential — verified by withMcpAuth in app/[transport]/route.ts before any handler runs. |
| MCP handler → platform-api.quiqup.com | Egress over TLS with `Authorization: Bearer <session-JWT>` minted via the Clerk → Quiqup actor-token bridge (`getQuiqupReadyJwt`). Anonymous calls are impossible — handlers throw when auth.userId is null. |
| Tool args (LLM-supplied) → upstream path/query/body | LLM-controlled values reaching upstream: `order_uuid` (path), 7 query params on list_integration_order_reasons (all bounded by Zod), repair-orders body (ids array capped at 50, source enum-bounded, dates as strings). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | All 5 shared-integration handlers | mitigate | Every handler refuses to call upstream when `auth.userId` is null (throws `Error("<tool> requires an authenticated user")`). The Clerk → Quiqup actor-token exchange is the single binding; no caller-supplied identity reaches upstream. Locked in by the "missing auth.userId throws" assertion in each test describe. |
| T-02-02 | Tampering | get_integration_order path injection | mitigate | `order_uuid` is encoded via `encodeURIComponent` before URL interpolation. Zod `z.string().min(1)` rejects empty strings at the schema layer. Test asserts a special-char uuid round-trips encoded. |
| T-02-03 | Tampering | list_integration_order_reasons query injection | mitigate | All 7 query params are built via URLSearchParams (not string concatenation) — URLSearchParams handles encoding deterministically and prevents query-string injection. limit is z.number().int().min(1).max(200); offset is z.number().int().min(0) — caps the response size to bound upstream cost. |
| T-02-04 | Tampering | repair_integration_orders mass-repair abuse | mitigate | ids array is z.array(z.string()).min(1).max(50) — caps batch size to 50. `source` is an explicit enum (shopify/woocommerce/salla) — no free-form source values. guardrails.rateLimit caps the call rate at 5/minute. idempotency_key + 15-min TTL makes duplicate retries safe. audit:true emits a structured record on every call. |
