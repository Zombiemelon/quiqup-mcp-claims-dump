## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → Next.js MCP route | Clerk OAuth verified by withMcpAuth before any handler runs. |
| MCP handler → platform-api.quiqup.com | Bearer JWT via Clerk → Quiqup actor-token bridge. ALL 6 handlers use getQuiqupReadyJwt(auth.userId). |
| Tool args (LLM-supplied) → upstream path/query/body | shop_name (path + query), code (OAuth secret, query + body), token (Shopify access token, body), delivery_methods/locations arrays (body). |
| LLM-supplied OAuth code → upstream | The `code` param on setup_shopify_callback and update_shopify_connection is the Shopify OAuth authorization code. Single-use; upstream rejects replays with 422. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Spoofing | All 6 Shopify handlers | mitigate | Every handler throws on `!auth.userId`. Single binding via getQuiqupReadyJwt; no caller-supplied identity reaches upstream. Test suite's "missing auth.userId throws" assertion per describe locks this in. |
| T-02-11 | Tampering | get_shopify_config path injection | mitigate | shop_name is encoded via encodeURIComponent before URL interpolation. Zod z.string().min(1) rejects empty strings at the schema layer. Test asserts a special-char shop_name round-trips encoded. |
| T-02-12 | Information Disclosure | update_shopify_connection.token leakage | mitigate | (a) The audit middleware (lib/middleware/audit.ts) redacts keys named `token` via ALWAYS_REDACT_KEYS — body field `token` is redacted before the audit-record write.
