## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → Next.js MCP route | Clerk OAuth verified by withMcpAuth before any handler runs. |
| MCP handler → platform-api.quiqup.com | Bearer JWT via Clerk → Quiqup actor-token bridge. ALL 6 handlers use getQuiqupReadyJwt(auth.userId). |
| Tool args (LLM-supplied) → upstream path/query/body | site_name (path), site_url (query + body — z.string().url() bounded), token (WooCommerce REST consumer-secret — SENSITIVE), country_filter (ISO-3166 alpha-2 z.string().length(2) bounded), delivery_method/states arrays (body). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-19 | Spoofing | All 6 WooCommerce handlers | mitigate | Every handler throws on `!auth.userId`. Single binding via getQuiqupReadyJwt. Test suite's "missing auth.userId throws" assertion per describe locks this in. |
| T-02-20 | Tampering | get_woocommerce_config path injection | mitigate | site_name is encoded via encodeURIComponent. Zod z.string().min(1) rejects empty strings. Test asserts a special-char site_name round-trips encoded. |
| T-02-21 | Tampering | list_woocommerce_shipping_lines SSRF via site_url | mitigate | site_url is z.string().url() — Zod enforces URL shape at schema layer. The query value is appended via URLSearchParams (not raw concatenation). The endpoint does NOT cause Quiqup to fetch the URL — site_url is a lookup key against Quiqup's saved connections, not a fetch target. SSRF is structurally impossible here. |
| T-02-22 | Information Disclosure | setup_woocommerce_connection.token leakage | mitigate | (a) Audit middleware redacts keys named `token` via ALWAYS_REDACT_KEYS. (b) Tool description marks `token` as SENSITIVE. (c) Description-quality test grep-locks the wording. (d) audit:true emits a record so any caller pattern that includes the token is traceable. |
