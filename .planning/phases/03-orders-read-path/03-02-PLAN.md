## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → Next.js MCP route | Untrusted inbound. Clerk OAuth at+jwt is the only credential — verified by withMcpAuth in app/[transport]/route.ts before any handler runs. |
| MCP handler → api.quiqup.com (Quiqup REST) | Egress over TLS with `Authorization: Bearer <session-JWT>` minted via the Clerk → Quiqup actor-token bridge. |
| MCP handler → audit.quiqup.com (Audit) | Egress over TLS with NO Authorization header by upstream design. Authentication is enforced at the MCP transport boundary only. |
| Tool args (LLM-supplied) → upstream path/query | LLM-controlled values reaching upstream: `order_id` (path-encoded for Quiqup REST), `order_uuid` (z.string().uuid()-validated for Audit). |
