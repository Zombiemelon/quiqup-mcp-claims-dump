## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → Next.js MCP route | Untrusted inbound. Clerk OAuth at+jwt is the only credential. |
| MCP handler → platform-api.quiqup.com | Egress over TLS with `Authorization: Bearer <session-JWT>` minted via the Clerk → Quiqup actor-token bridge. |
| Tool args (LLM-supplied) → upstream query | LLM-controlled values reaching upstream: `value`, `intention`, `region`, `main_depot`. All built via URLSearchParams. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-17 | Spoofing | All 3 handlers | mitigate | Each refuses to run when `auth.userId` is null. Locked in by per-tool "rejects unauthenticated callers" tests. |
| T-03-18 | Tampering | Query-string injection | mitigate | URLSearchParams used in every handler (T-03-03 hygiene). Asserted in the per-tool "forwards X as query param" tests. Boolean values serialized via `String(args.main_depot)` to "true"/"false" — no toString() ambiguity. |
| T-03-19 | Tampering | `intention` free-form vs enum | accept | `intention` is `z.string().min(1)` (NOT a z.enum). Rationale: the upstream BE may add new transitions; over-constraining the client would silently break new intentions. The description lists the observed values; the BE returns the structured `error` envelope on bad ones (200-with-error contract). |
