<objective>
Add Langfuse eval coverage for every new service-host family introduced in Phase 3:
  - **Orders Core GraphQL** (anchored on lookup_orders_ids + bulk_orders_lookup from plan 03-01)
  - **Quiqup REST + Audit** (combined, anchored on get_order_history + list_order_audit_events from plan 03-02 — grouped because both are order-detail reads with shared prompts, mirroring the Phase-2 sub-family grouping rationale)
  - **Ex-core** (anchored on download_orders_export from plan 03-04)
  - **Orders Core REST** (anchored on upload_order_document from plan 03-04)

Each eval bundles the standard scorers (tool-name-match, required-fields-present, args-overlap, description-quality) and adds Phase-3-specific STATIC source-inspection scorers that lock in the auth-exception, binary-envelope, and server-binding invariants established in Waves 1-4.

Purpose: PROJECT.md's stated invariant — "Every new service-host family gets at least one Langfuse eval before its tools count as shipped" — drives this wave. Phase 3 introduces 4 new service hosts and the eval gate must extend to cover them. Without this, regressions on the auth-exception (audit.ts gaining a Bearer header), the binary contract (download_orders_export reverting to text-only), or the server-binding (upload_order_document accepting user_id) would land silently.

Output: 4 new eval runners, 4 new datasets, 4 new score files, 4 new `pnpm eval:*` scripts, and 4 new CI workflow jobs.
</objective>

<!-- Langfuse experiment pattern (from evals/get-account.ts and evals/score-get-account.ts):
       - Dataset = `items` array, each item shape `{ input: { request: string }, expected: <ExpectedShape> }`.
       - Task function: takes an item, builds Anthropic SDK request with `tools: [tool]` (where `tool` is constructed from `spec.name, spec.description, zodToJsonSchema(spec.inputSchema)`), invokes, returns `{ chosen_tool, chosen_args, raw_completion, description }`.
       - `evaluators` = array of named scorer functions `(item, output) => { name, value }`.
       - `langfuse.experiment.run({ name, description, data: items, task, evaluators })` runs the suite.
       - `EVAL_DRY_RUN=1` → short-circuits to a print-and-exit path (CI uses this).
       - `EVAL_GATE=1` → invokes `gate(result, [{ scoreName, min }, ...])` at the end (fails process on regression).
       - Anthropic SDK loaded via dynamic import; OTel SDK + LangfuseSpanProcessor instrument the run.
