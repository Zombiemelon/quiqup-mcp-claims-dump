quiqup-mcp-claims-dump — End-to-End Flow

From a user opening Claude.ai to a Quiqup order rendered in chat. Each phase is one Mermaid diagram. Function-by-function, with file:line refs back into the codebase.

💡 Hover any node in the diagrams below — every step has a detailed tooltip explaining the why or gotcha behind it.

event (start/end) task / function call gateway (decision) state / store failure path

Five phases. Phase A (OAuth dance) runs once per device. Phases B → E run on every tool call.

Two trust boundaries. Inbound = Clerk validates the user (B). Outbound = MCP authenticates itself to Quiqup (C, D). The user identity never crosses the second boundary — that's the BFF pattern.

Phase E — Project, return, render

Raw Quiqup payload is fat (~150 lines per order). The tool projects ~10 fields per row and wraps it in a summary. mcp-handler streams it back to Claude.ai.

🗄️ Storage layers menu — what CAN be stored on the MCP

All of these are available to a Vercel Lambda. Currently we only use one (module-scoped in-memory). Ranked by friction and durability.

Storage typeDurabilityLatencyTypical use
Module-scoped variable (current)Lost on cold start (~minutes)0msShort-lived tokens, JWKS cache
Vercel KV (Redis on Upstash)Durable<10msToken persistence across cold starts, per-user merchant mappings, rate-limit counters
Vercel Edge ConfigDurable, write-slow<5ms reads, seconds writesRead-mostly config, NOT for hourly-rotating tokens
Clerk publicMetadata / privateMetadataDurable, lives at Clerk~30ms (round-trip)User-attached config (e.g. quiqup_partner_id per user)
External Postgres (Neon, Supabase)Durable, queryable10-50msAudit logs, impersonation history, full state
Vercel BlobDurable50-200msFiles, NOT credentials

🧠 Mental model: the MCP server should hold its own credentials (Quiqup OAuth client, possibly KV for caching). It should NOT hold user credentials — those are evidence-of-identity that pass through, not state to retain. The inbound trust domain holds user data; the MCP holds bridge state; the outbound trust domain holds the data itself.
